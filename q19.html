🔤 Вопрос 19. Типографика в UI

Типы шрифтов (serif, sans-serif, моноширинные) и критические параметры читаемости: размер, длина строки, интерлиньяж, контраст.

🔤 Основные типы шрифтов

1. Serif (с засечками)

Пример: Times New Roman, Georgia, Merriweather. У букв есть декоративные засечки. Классические, «книжные», внушают доверие. На экране при малых размерах читаемость может снижаться.

Где использовать: длинные статьи, блоги, заголовки (элегантность), не рекомендуется для мелкого текста (подписи) и дашбордов.

2. Sans-serif (без засечек)

Пример: Inter, Roboto, Helvetica, Arial. Чистые линии, максимально читаемые на экранах. Стандарт де-факто для UI.

Где использовать: почти везде: основной текст, кнопки, поля ввода, навигация, мобильные интерфейсы, дашборды, CRM.

3. Моноширинные (Monospace)

Пример: JetBrains Mono, Fira Code, Consolas. Каждый символ одинаковой ширины. Технические, ассоциируются с кодом.

Где использовать: блоки кода, терминалы, логи, таблицы с колонками, поля ввода паролей. Не использовать для абзацев текста.

📏 Критические параметры читаемости

1. Размер шрифта (font size)

  • Десктоп: основной текст ≥16px (или 1rem). Вспомогательный (подписи) — 14px, но не для длительного чтения.
  • Мобильные: 16px (Apple рекомендует минимум 16px для полей ввода, чтобы избежать зума).
  • Заголовки: 1.5–2× от основного (24–32px).

2. Длина строки (line length / measure)

  • Десктоп: 50–75 символов на строку (включая пробелы).
  • Мобильные: 35–50 символов.
  • Контроль: max-width: 600px для текста (при 16px даст ~60–70 символов).

3. Интерлиньяж (line height)

  • Оптимально: 1.4–1.6 от размера шрифта (без единиц).
  • Пример: line-height: 1.5 при 16px = 24px.
  • Для заголовков — 1.2–1.3; для мобильных и длинных текстов — ближе к 1.6.

4. Контраст (contrast)

WCAG 2.1 уровень AA: обычный текст — коэффициент контраста ≥4.5:1; крупный текст (≥18pt или 14pt жирного) — ≥3:1.

  • Плохо: серый #AAAAAA на белом (~2.3:1) — не читается.
  • Хорошо: #333333 на белом (~12:1), #595959 (~7:1).
  • Инструменты: WebAIM, Lighthouse, axe DevTools.
💡 Практика: для основного текста — чёрный или очень тёмно-серый на светлом фоне; для второстепенного — серый, но с контрастом ≥4.5:1.

📌 Дополнительные параметры (менее критичны)

ПараметрРекомендация
Трекинг (letter-spacing)Для основного текста — 0. Для заголовков заглавными можно увеличить (0.05em).
Регистр (uppercase)Избегать длинных фраз — читаются на 20% медленнее. Только для коротких акронимов или меток.
ВыравниваниеОсновной текст — по левому краю (без полного выравнивания, чтобы избежать «речных» дыр).

🎯 Золотая формула: хорошая типографика = правильный выбор шрифта + достаточный размер + комфортная длина строки + просторный интерлиньяж + высокий контраст.
