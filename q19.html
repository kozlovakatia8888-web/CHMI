🔤 Типографика в UI — углублённо

Типы шрифтов (serif, sans-serif, моноширинные) и критические параметры читаемости: размер, длина строки, интерлиньяж, контраст.

Типы шрифтов

  • Serif (с засечками): Times New Roman, Georgia, Merriweather. Классические, «книжные», внушают доверие. На экране при малых размерах могут терять читаемость. Используются для длинных статей, заголовков, брендовых текстов.
  • Sans-serif (без засечек): Inter, Roboto, Helvetica, Arial. Современные, нейтральные, максимально читаемые на экранах. Стандарт де-факто для UI: кнопки, поля, навигация, таблицы.
  • Monospace (моноширинные): Courier New, Consolas, JetBrains Mono. Каждый символ одинаковой ширины. Технические, ассоциируются с кодом. Используются для блоков кода, терминалов, таблиц с колонками. Не использовать для абзацев.
📌 Рекомендация: для основных интерфейсов выбирайте sans-serif. Serif — для акцентов и длинных чтений. Monospace — только для технического контента.

📏 Критические параметры читаемости

ПараметрРекомендацияПочему важно
Размер (font size)Основной текст ≥16px (десктоп), 16px (мобильные). Заголовки 24–32px.Слишком мелко — напряжение зрения, слишком крупно — много прокрутки.
Длина строки (line length)50–75 символов (десктоп), 35–50 (мобильные). Контроль через max-width.Слишком длинная — теряется ритм; слишком короткая — частые переносы.
Интерлиньяж (line height)1.4–1.6 от размера шрифта. Пример: line-height: 1.5.Малый — строки слипаются; большой — теряется связность.
Контраст (contrast)WCAG AA: обычный текст ≥4.5:1. Плохо: #AAAAAA на белом (2.3:1).Низкий контраст — нечитаемо для слабовидящих и при плохом освещении.
🔧 Дополнительно: избегайте длинных фраз в uppercase (читаются на 20% медленнее). Трекинг по умолчанию 0, для заглавных можно 0.05em. Выравнивание по левому краю (не по ширине).
