🛠️ Инструментарий бэкенд-разработки: Middleware и Dependency Injection (DI)

Конвейер обработки запросов и внедрение зависимостей для тестируемости и гибкости.

Middleware и pipeline

Функции, обрабатывающие запрос до/после основного обработчика. Порядок важен.

// Express.js app.use(logger); app.use(authentication); app.use(compression); app.get('/', handler);

Схема: req → [logger] → [auth] → [compression] → handler → [compression] → [auth] → [logger] → res

  • Типовые задачи: логирование, аутентификация, CORS, парсинг тела, rate limiting.
  • Примеры: Django (MIDDLEWARE), Spring (Filter), ASP.NET Core (app.Use).

Dependency Injection (DI)

Объект получает зависимости извне (конструктор), а не создаёт их сам через new. Управление зависимостями выносится в DI-контейнер.

// Плохо (без DI) class UserService { constructor() { this.db = new MySQLConnection(); // жёсткая связь } } // Хорошо (DI) class UserService { constructor(db, logger) { this.db = db; this.logger = logger; } } const service = new UserService(new PostgreSQLConnection(), new ConsoleLogger());
  • ✅ Преимущества: тестируемость (mock), гибкость (смена реализации), слабая связанность.
  • ✅ DI-контейнеры: Spring (@Autowired), ASP.NET Core (IServiceCollection), FastAPI (Depends).
💡 Антипаттерн Service Locator — скрывает зависимости, усложняет тестирование. DI предпочтительнее.
