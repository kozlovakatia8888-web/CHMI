🛠️ Инструментарий бэкенд-разработки: Middleware и Dependency Injection (DI)

Конвейер обработки запросов и внедрение зависимостей для тестируемости и гибкости.

🔁 Middleware и pipeline

Middleware — функции (или классы), которые участвуют в обработке входящего HTTP-запроса и исходящего ответа. Они вызываются последовательно, образуя конвейер (pipeline).

// Express.js app.use(logger); // middleware 1 app.use(authentication); // middleware 2 app.use(compression); // middleware 3 app.get('/', handler); // терминальный обработчик

Порядок вызова: req → [m1] → [m2] → [m3] → handler → [m3] → [m2] → [m1] → res

  • Типовые задачи: логирование, аутентификация/авторизация, CORS, парсинг тела (JSON/URL-encoded), сжатие (gzip), rate limiting, CSRF, трассировка.
  • Примеры в фреймворках: Django (MIDDLEWARE), Spring (Filter, HandlerInterceptor), ASP.NET Core (app.Use()), FastAPI (@app.middleware("http")).
💡 Важно: порядок регистрации middleware определяет порядок выполнения. Ошибка в порядке может привести к некорректной работе (например, аутентификация до парсинга тела).

💉 Dependency Injection (DI)

Суть: объект получает зависимости извне (через конструктор, сеттер), а не создаёт их сам через new. Управление зависимостями выносится в DI-контейнер.

// Плохо (без DI) — жёсткая связь class UserService { constructor() { this.db = new MySQLConnection(); // нельзя подменить для теста } } // Хорошо (DI) — зависимости передаются class UserService { constructor(db, logger) { this.db = db; this.logger = logger; } } // Снаружи const service = new UserService(new PostgreSQLConnection(), new ConsoleLogger());
  • ✅ Преимущества: тестируемость (подмена на mock), гибкость (смена реализации без изменения класса), слабая связанность, переиспользование.
  • ✅ DI-контейнеры: Spring (Java) — @Autowired, ASP.NET Core — IServiceCollection, FastAPI (Python) — Depends, Angular (фронтенд) — встроенный DI.
⚠️ Антипаттерн Service Locator (например, ServiceLocator.get('db')) — скрывает зависимости, усложняет тестирование. DI предпочтительнее.
