📄 Вопрос 2. Модель DOM и события

Фазы захвата и всплытия, делегирование событий — полное объяснение с примерами и тестом.

🌳 Что такое DOM?

DOM (Document Object Model) — это программный интерфейс, который браузер строит из HTML-кода. Это дерево узлов, где каждый узел — это элемент, атрибут, текст. JavaScript может изменять DOM в реальном времени.

// Пример: получаем элемент и меняем текст document.getElementById('myTitle').textContent = 'Новый заголовок';
💡 Браузер предоставляет API: document.querySelector, addEventListener, createElement и другие.

🎯 Событийная модель: захват, цель, всплытие

Когда пользователь кликает, событие проходит три фазы:

  1. Захват (capture) — от корня (window) до целевого элемента.
  2. Цель (target) — событие достигает элемента, на котором произошло действие.
  3. Всплытие (bubbling) — обратный путь от цели к корню.
// Обработчик на фазе всплытия (по умолчанию) element.addEventListener('click', handler); // Обработчик на фазе захвата element.addEventListener('click', handler, { capture: true });
📌 Пример: клик по кнопке внутри div. Сначала capture на div, потом на кнопке, потом bubbling на div.

🧩 Делегирование событий (Event Delegation)

Суть: Один обработчик на родителе, который ловит события от всех дочерних элементов благодаря всплытию. Это решает проблему динамически добавленных элементов и экономит память.

// Плохо: обработчик на каждой кнопке document.querySelectorAll('.delete').forEach(btn => btn.addEventListener(...)); // Хорошо: делегирование document.getElementById('list').addEventListener('click', (e) => { if (e.target.classList.contains('delete')) { e.target.remove(); } });

Задачи, которые решает делегирование: работа с динамическими элементами, экономия памяти, централизованная логика, упрощение кода.

⚠️ Не все события всплывают (например, focus). Для них используйте focusin/focusout.

📋 Сводная таблица

| Термин | Суть | |-------------------|------| | DOM | Дерево HTML-элементов, доступное через JS | | Фаза захвата | Событие идёт сверху вниз (от window к цели) | | Фаза цели | Событие на элементе, который вызвал действие | | Фаза всплытия | Событие идёт снизу вверх (от цели к window) | | Делегирование | Один обработчик на родителе ловит события от детей |
