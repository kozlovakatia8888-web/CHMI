🌳 Модель DOM и события: фазы, всплытие, захват, делегирование

Глубокое объяснение с примерами, схемами и глоссарием.

📌 Что такое DOM

DOM (Document Object Model) — это программный интерфейс, который браузер строит из HTML-кода при загрузке страницы. Каждый тег, атрибут, текст — это узел (node) в древовидной структуре.

DOM предоставляет API для изменения структуры, стилей и содержимого в реальном времени:

  • document.getElementById('id') — поиск по идентификатору.
  • document.querySelector('.class') — поиск по CSS-селектору.
  • element.textContent = 'Новый текст' — изменение содержимого.
  • element.style.color = 'red' — изменение стиля.
// Пример манипуляции DOM const header = document.getElementById('main-header'); header.textContent = 'Привет, мир!'; header.style.backgroundColor = '#333';
💡 DOM не является частью JavaScript — это интерфейс, предоставляемый браузером (Web API).

🎯 Событийная модель: захват, цель, всплытие

Когда происходит событие (клик, нажатие клавиши), браузер создаёт объект события и проводит его по дереву в три этапа:

  1. Capture (захват) — от корня window → document → <html> → ... к целевому элементу. Обработчики, установленные с { capture: true }, срабатывают на этом этапе.
  2. Target (цель) — событие достигает элемента, на котором произошло действие.
  3. Bubbling (всплытие) — событие поднимается от целевого элемента обратно к корню. По умолчанию обработчики срабатывают именно на фазе всплытия.
// Пример: обработчики на родителе и потомке const parent = document.getElementById('parent'); const child = document.getElementById('child'); parent.addEventListener('click', () => console.log('parent (bubble)')); child.addEventListener('click', () => console.log('child')); parent.addEventListener('click', () => console.log('parent capture'), { capture: true }); // При клике на child порядок: parent capture → child → parent bubble
⚠️ Остановка всплытия: event.stopPropagation(). Используйте осторожно, чтобы не нарушить делегирование.

Когда это важно: понимание фаз помогает точно управлять порядком обработки событий, особенно в сложных интерфейсах с вложенными элементами.

🧩 Паттерн «Делегирование событий» (Event Delegation)

Суть: вместо того чтобы вешать обработчик на каждый динамический элемент, мы вешаем один обработчик на общего родителя и используем всплытие. Это решает две ключевые задачи:

  • Экономия памяти (один обработчик вместо сотен).
  • Поддержка динамически добавляемых элементов (новые элементы автоматически получают обработку).
// Плохо (без делегирования) document.querySelectorAll('.delete-btn').forEach(btn => { btn.addEventListener('click', () => btn.remove()); }); // Новые кнопки, добавленные позже, не будут работать. // Хорошо (делегирование) document.getElementById('list-container').addEventListener('click', (event) => { if (event.target.classList.contains('delete-btn')) { event.target.closest('.item')?.remove(); } });

Дополнительные возможности:

  • Можно обрабатывать несколько типов элементов через event.target.matches().
  • Удобно для списков, таблиц, динамических форм.
⚠️ Важно: некоторые события не всплывают (focus, blur, load, scroll, mouseenter). Для них существуют аналоги: focusin, focusout.
