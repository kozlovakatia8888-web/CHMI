📄 Вопрос 2. Модель DOM и события

Фазы захвата и всплытия, делегирование событий — как браузер обрабатывает клики и почему это важно для интерфейсов.

🌳 Что такое DOM (Document Object Model)

DOM — это программный интерфейс, который браузер строит из HTML-кода. HTML — чертёж, а DOM — уже собранный дом, где у каждой стены, двери и розетки есть свойства и методы, доступные через JavaScript.

  • Структура: дерево узлов. Корень — document, дальше <html>, <head>, <body>, вложенные теги, текст, атрибуты.
  • API: document.getElementById, querySelector, addEventListener — чтобы читать, изменять, удалять или добавлять узлы в реальном времени.
💡 Зачем знать? Любое взаимодействие с веб-страницей (изменение текста, анимация, обработка кликов) происходит через DOM.

🎯 Событийная модель: захват, цель, всплытие

Когда пользователь кликает или нажимает клавишу, браузер генерирует событие, которое проходит строгий путь по дереву DOM. Этот путь делится на три фазы:

  1. Фаза захвата (capture phase) — событие движется от корня window → document → <html> → … вниз к целевому элементу.
  2. Фаза цели (target phase) — событие достигает элемента, на котором произошло действие (например, кнопки).
  3. Фаза всплытия (bubbling phase) — событие поднимается обратно вверх от целевого элемента к корню.
// По умолчанию addEventListener слушает на фазе всплытия (useCapture = false) element.addEventListener('click', handler, false); // Чтобы поймать на фазе захвата — третий аргумент true или { capture: true } element.addEventListener('click', handler, { capture: true });
📌 Пример: если кликнуть по кнопке внутри двух вложенных div, событие сначала спустится (захват), достигнет кнопки (цель), затем поднимется (всплытие). Обработчики на родителях сработают во время всплытия, если не указан захват.

🧩 Паттерн «Делегирование событий» (Event Delegation)

Суть: не вешать обработчик на каждый динамический элемент, а повесить один обработчик на общего родителя и использовать всплытие, чтобы ловить события от дочерних элементов.

❌ Без делегирования (плохо)

document.querySelectorAll('.delete-btn').forEach(btn => { btn.addEventListener('click', () => btn.parentElement.remove()); }); // Проблемы: новые кнопки не получат обработчик; много слушателей.

✅ С делегированием (хорошо)

document.getElementById('list-container').addEventListener('click', (event) => { if (event.target.classList.contains('delete-btn')) { event.target.parentElement.remove(); } }); // Работает для любых кнопок, добавленных в будущем. Один обработчик.

🎯 Какие задачи решает делегирование?

  • Динамический контент — новые элементы сразу работают.
  • Экономия памяти — один родительский обработчик вместо сотен.
  • Упрощение кода — не нужно вручную навешивать и снимать обработчики.
  • Централизованная логика — удобно обрабатывать разные типы элементов (кнопки, чекбоксы) внутри одного родителя.

⚠️ Важные нюансы

  • Не все события всплывают: focus, blur, load, scroll, mouseenter/mouseleave — не всплывают. Для них есть аналоги (focusin/focusout).
  • В обработчике нужно точно идентифицировать целевой элемент через event.target. Иногда приходится подниматься по дереву (closest), если кликнули на вложенный элемент (например, <span> внутри кнопки).
  • event.stopPropagation() останавливает всплытие — используйте осторожно, это может сломать делегирование на вышестоящих родителях.

📋 Итог: ключевые понятия

ТерминСуть
DOMДревовидное представление HTML-документа, доступное для JS.
Фаза захватаСобытие идёт сверху вниз (от window к цели).
Фаза целиСобытие достигло элемента, на котором произошло действие.
Фаза всплытияСобытие поднимается от цели к корню.
ДелегированиеОдин обработчик на родителе, ловит события от всех детей за счёт всплытия.
Решаемые задачиДинамические элементы, экономия памяти, чистота кода.
💬 Вопросы на экзамене: «Как остановить всплытие?» → event.stopPropagation(). «Чем отличается target от currentTarget?» → target — реальный элемент-источник события, currentTarget — элемент, на котором висит обработчик (в делегировании это родитель).

🔁 Пример для наглядности

<div id="outer"> <div id="inner"> <button id="btn">Кликни</button> </div> </div> // При клике на кнопку порядок фаз: // Захват: window → #outer → #inner → #btn // Цель: #btn // Всплытие: #btn → #inner → #outer → … → window

Если повесить обработчик на #outer на фазе всплытия — он сработает после обработчика кнопки. Если на фазе захвата — до.
