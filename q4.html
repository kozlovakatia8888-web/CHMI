🧠 Вопрос 4. Модель GOMS и KLM

Предсказание времени и эффективности интерфейса — количественные методы HCI для сравнения дизайнов до тестирования.

🎯 GOMS (Goals, Operators, Methods, Selection rules)

GOMS — аналитическая модель, описывающая решение задачи пользователем через когнитивные и моторные действия.

КомпонентЧто означаетПример (отправить email)
Goals (Цели)Что пользователь хочет достичь. Иерархичны.Отправить письмо → Ввести адрес → Ввести тему → Нажать отправить
Operators (Операторы)Элементарные действия (моторные, перцептивные, когнитивные).Переместить мышь, кликнуть, напечатать текст
Methods (Методы)Последовательность операторов для достижения цели.Метод 1: клик мыши по кнопке «Отправить»
Метод 2: нажать Ctrl+Enter
Selection rules (Правила выбора)Условия выбора метода.Если поле активно → Ctrl+Enter, иначе → мышь

Для чего используется GOMS? Сравнение дизайнов (быстрее/медленнее), прогнозирование времени обучения, выявление лишних шагов, обоснование решений цифрами. Ограничение: сложен для больших систем, предполагает экспертного пользователя (без ошибок).

⚡ KLM (Keystroke-Level Model)

Упрощённая версия GOMS. Оцениваем время задачи, складывая время на примитивные физические и ментальные действия с фиксированными константами.

ОператорОбозначениеОписаниеВремя (с)
KeystrokeKНажатие клавиши (опытный пользователь)0.20
PointPНаведение мыши на цель1.10
Button pressBНажатие кнопки мыши0.10
Double clickBBДвойной клик0.20
HandHПеренос руки с клавиатуры на мышь или обратно0.40
MentalMМентальная подготовка, обдумывание, поиск1.35
ResponseRВремя отклика системызависит

Правила KLM:

  • Добавлять M перед каждым значимым когнитивным действием.
  • Если группа операторов образует «когнитивно единое» действие, M ставится один раз.
  • P и B всегда идут вместе при клике: P + B = 1.2 с.
  • Время выполнения задачи = сумма операторов.

🌡️ Практический пример: конвертер температур (из лекции)

Интерфейс: поле ввода °C, кнопка «Конвертировать», поле вывода °F.

❌ Вариант А (плохой дизайн — только по кнопке)

M (найти поле) 1.35 + H (рука на мышь) 0.4 + P 1.1 + B 0.1 + H (рука на клаву) 0.4 + KKK (ввод "100") 0.6 + H 0.4 + P 1.1 + B 0.1 + M (посмотреть на результат) 1.35

≈ 6.9 секунд

✅ Вариант Б (хороший дизайн — конвертация в реальном времени, фокус в поле)

M (посмотреть на поле) 1.35 + KKK (ввод "100") 0.6 + M (посмотреть на результат) 1.35

≈ 3.3 секунды — выигрыш в 2 раза

💡 Вывод: убраны операторы H, P, B и лишний M — время сократилось вдвое.

📌 Практические выводы из KLM для дизайнера

  • Сокращайте число ментальных операторов (M) — группируйте действия, делайте интерфейс предсказуемым.
  • Убирайте лишние клики (P+B) — живой поиск, автодополнение, автосохранение вместо кнопок «Обновить».
  • Уменьшайте переключения между клавиатурой и мышью (H) — горячие клавиши для частых действий.
  • Минимизируйте расстояние указателя (P) — закон Фиттса: крупные кнопки, близко к курсору.
  • Автоматизируйте то, что можно предсказать — в конвертере результат вычисляется из ввода, кнопка не нужна.

📊 Итог: GOMS vs KLM

МодельНазначениеУровень детализацииВыходные данные
GOMSПолный когнитивно-моторный анализЦели, операторы, методы, правила выбораПредсказание времени, сравнение методов, оценка обучения
KLMУпрощённое предсказание времениКлавиши, указание, нажатие, перемещение руки, ментальные операцииЧисловое время выполнения задачи (сумма констант)
🎯 KLM особенно полезна для сравнения двух альтернатив: выбираем ту, у которой суммарное время меньше. Относительное сравнение — очень точное.
