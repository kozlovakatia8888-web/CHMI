🧠 Модель GOMS и KLM — предсказание времени и эффективности интерфейса

Когнитивные модели для сравнения альтернативных дизайнов. Глубокий разбор с примерами и глоссарием.

🎯 GOMS (Goals, Operators, Methods, Selection rules)

GOMS — аналитическая модель, описывающая решение задачи пользователем через когнитивные и моторные действия. Компоненты:

  • Goals (Цели) — что пользователь хочет достичь (иерархически).
  • Operators (Операторы) — элементарные действия (нажатие клавиши, перемещение мыши, ментальные операции).
  • Methods (Методы) — последовательности операторов для достижения цели (несколько методов для одной цели).
  • Selection rules (Правила выбора) — условия, по которым пользователь выбирает метод.

Пример (отправить email): Цель → Отправить письмо. Метод 1: клик мыши по кнопке «Отправить». Метод 2: нажать Ctrl+Enter. Правило выбора: если поле активно → Ctrl+Enter, иначе → мышь.

GOMS используется для сравнения дизайнов, прогнозирования времени обучения и выявления лишних шагов. Ограничение: сложен для больших систем, предполагает экспертного пользователя.

⚡ KLM (Keystroke-Level Model) — упрощённая версия GOMS

Принцип: сумма времени на примитивные операции (клавиши, указание, нажатие, ментальные операции). Базовые операторы:

ОператорОбозначениеВремя (с)Описание
KeystrokeK0.20Нажатие клавиши
PointP1.10Наведение мыши
Button pressB0.10Нажатие кнопки мыши
HandH0.40Перенос руки между клавиатурой и мышью
MentalM1.35Ментальная подготовка, обдумывание
ResponseRзависитВремя отклика системы

Практический пример: конвертер температур

Вариант А (плохой дизайн — с кнопкой): M (найти поле) + H + P + B + H + KKK (ввод) + H + P + B + M (посмотреть на результат) ≈ 6.9 с.

Вариант Б (хороший дизайн — без кнопки): M + KKK + M ≈ 3.3 с. Выигрыш в 2 раза.

💡 Вывод: убраны лишние H, P, B и сокращено число M. KLM позволяет сравнивать альтернативы до реализации.
