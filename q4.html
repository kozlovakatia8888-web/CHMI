🧠 GOMS и KLM: предсказание времени и эффективности интерфейса

Количественные методы HCI для сравнения дизайнов до тестирования: компоненты GOMS, операторы KLM и расчёт времени на примере конвертера температур.

GOMS (Goals, Operators, Methods, Selection rules)

Аналитическая модель, описывающая решение задачи пользователем через когнитивные и моторные действия.

КомпонентЧто означаетПример (отправить email)
GoalsЦели (иерархичны)Отправить письмо → Ввести адрес → Нажать отправить
OperatorsЭлементарные действия (моторные, перцептивные, когнитивные)Переместить мышь, кликнуть, напечатать текст
MethodsПоследовательности операторов для достижения целиМетод 1: клик мыши по кнопке «Отправить»; Метод 2: Ctrl+Enter
Selection rulesУсловия выбора методаЕсли поле активно → Ctrl+Enter, иначе → мышь
🔍 GOMS позволяет сравнивать альтернативные дизайны (быстрее/медленнее), прогнозировать время обучения, выявлять лишние шаги. Ограничение: сложен для больших систем, предполагает экспертного пользователя.

⚡ KLM (Keystroke-Level Model) — упрощённая версия GOMS

Оценивает время выполнения задачи, суммируя константы для примитивных действий.

ОператорОбозначениеОписаниеВремя (с)
KeystrokeKНажатие клавиши (опытный пользователь)0.20
PointPНаведение мыши на цель1.10
Button pressBНажатие кнопки мыши0.10
HandHПеренос руки с клавиатуры на мышь или обратно0.40
MentalMМентальная подготовка, поиск в интерфейсе1.35
ResponseRВремя отклика системы (обычно 0)зависит
// Правила KLM: // - M ставится перед каждым значимым когнитивным действием. // - Если группа операторов образует единое действие — M один раз. // - P и B идут вместе: P + B = 1.2 с.

🌡️ Пример: конвертер температур (из лекции)

Вариант А (плохой — кнопка «Конвертировать»): M (найти поле) 1.35 + H 0.4 + P 1.1 + B 0.1 + H 0.4 + KKK (ввод "100") 0.6 + H 0.4 + P 1.1 + B 0.1 + M (посмотреть на результат) 1.35 = ≈ 6.9 с

Вариант Б (хороший — конвертация в реальном времени, фокус в поле): M (посмотреть на поле) 1.35 + KKK (ввод "100") 0.6 + M (посмотреть на результат) 1.35 = ≈ 3.3 с

💰 Выигрыш в 2 раза за счёт удаления H, P, B и лишнего M. Вывод: не нужны подтверждающие кнопки, когда результат однозначно вычисляется из ввода.

📌 Итог: GOMS vs KLM

МодельНазначениеУровень детализацииВыходные данные
GOMSПолный когнитивно-моторный анализЦели, операторы, методы, правила выбораПредсказание времени, сравнение методов, оценка обучения
KLMУпрощённое предсказание времениКлавиши, указание, нажатие, ментальные операцииЧисловое время выполнения задачи
💡 KLM особенно полезна для сравнения двух альтернатив: выбираем ту, у которой суммарное время меньше. Относительное сравнение — очень точное.
