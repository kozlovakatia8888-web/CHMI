🎨 Вопрос 13. Дизайн-системы и доступность (WCAG)

Что такое дизайн-система (Material, HIG), её компоненты. Принципы веб-доступности: контрастность, скринридеры, клавиатурная навигация.

🧩 Что такое дизайн-система?

Дизайн-система — единый набор правил, компонентов, паттернов и гайдлайнов, обеспечивающий визуальную и функциональную согласованность продуктов компании. Примеры: Material Design (Google), Human Interface Guidelines (HIG) (Apple), Fluent (Microsoft), Ant Design.

КомпонентОписаниеПример
Токены (design tokens)Базовые атомарные значения: цвета, типографика, отступы, тени.color-primary: #0052CC, spacing-small: 8px
UI-компоненты Переиспользуемые элементы: кнопки, поля ввода, карточки, модалки. Button (primary, secondary, disabled), Input, Modal
Паттерны Типовые решения для часто встречающихся задач. Форма логина, фильтрация списка, поиск с автодополнением.
Гайдлайны и принципы Инструкции по использованию компонентов, текстам, обработке ошибок. «Не используйте иконку без подписи для важных действий».
Документация и песочница Живой сайт (Storybook, Zeroheight) с примерами и кодом. Storybook с аддонами для accessibility.
💡 Зачем нужна дизайн-система? Ускоряет разработку, обеспечивает единый UX, упрощает поддержку, позволяет внедрять доступность централизованно.

♿ Веб-доступность (a11y) и стандарт WCAG

WCAG (Web Content Accessibility Guidelines) — международный стандарт W3C. Актуальная версия WCAG 2.1 (уровни A, AA, AAA). Законодательство обычно требует уровня AA.

Четыре принципа WCAG (POUR):

ПринципСутьПример
Perceivable (Воспринимаемость)Информация должна быть представлена так, чтобы пользователь мог её увидеть/услышать.Альт‑текст для картинок, субтитры к видео, достаточная контрастность.
Operable (Управляемость)Интерфейсом можно управлять с клавиатуры, жестов; достаточно времени на ввод.Навигация по Tab, видимый фокус, кнопка «отмена» для каруселей.
Understandable (Понятность)Интерфейс предсказуем, язык прост, ошибки объясняются.Кнопка «Отправить» всегда отправляет; подсветка поля с ошибкой.
Robust (Надёжность)Код корректно интерпретируется браузерами и вспомогательными технологиями.Валидный HTML, правильные атрибуты ARIA, семантические теги.

🔧 Примеры реализации доступности

1. Контрастность

Требование WCAG 2.1 (уровень AA): для обычного текста — коэффициент контраста не менее 4.5:1; для крупного текста и элементов интерфейса — не менее 3:1.

  • Плохо: серый текст #AAAAAA на белом фоне (контраст ~2.3:1).
  • Хорошо: #333333 на белом (контраст ~12:1) или #595959 на белом (~7:1).
  • Инструменты проверки: WebAIM Contrast Checker, Lighthouse, axe DevTools.

2. Поддержка скринридеров (для слепых и незрячих)

  • Альтернативный текст — атрибут alt. Пустая строка для декоративных картинок.
  • Семантический HTML — <nav>, <main>, <h1>, <button> вместо <div onclick>.
  • ARIA-атрибуты (когда нет нативного тега): aria-label, aria-labelledby, aria-hidden="true", role="status".
✅ Хорошо: кнопка-иконка с пояснением для скринридера <button aria-label="Закрыть диалог">✖</button> ❌ Плохо: не семантично, скринридер не поймёт <div class="button" onclick="...">Нажми меня</div>

3. Навигация с клавиатуры (для людей с моторными нарушениями)

  • Логический порядок фокуса (Tab движется слева направо).
  • Видимый индикатор фокуса — никогда outline: none без замены на :focus-visible.
  • Обработка клавиш: Enter/Пробел (активация), Стрелки (меню, радиокнопки), Esc (закрыть модалку).
  • Skip link — скрытая ссылка для перехода к основному содержимому.
<a href="#maincontent" class="skip-link">Перейти к основному содержанию</a> /* CSS: .skip-link { position: absolute; left: -9999px; } .skip-link:focus { left: auto; } */

🎯 Главная мысль: Дизайн-система должна включать доступность как неотъемлемую часть (accessible by default), а не как доработку после релиза. WCAG — это не рекомендация, а требование законодательства во многих странах.
