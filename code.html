📘 Шпаргалка по коду: Бэкенд + Фронтенд

Быстрая навигация по файлам, функциям и ключевым механизмам проекта (FastAPI, SQLAlchemy, React, MUI).

⚙️ 1. Бэкенд (FastAPI + SQLAlchemy)

1.1. ORM (SQLAlchemy) — взаимодействие с базой

  • Модели — app/models.py (классы Brand, Category, Product, User, Order). Наследуются от Base (из database.py).
  • Сессии — app/database.py: engine, SessionLocal, функция get_db (зависимость).
  • CRUD — app/crud.py: функции create_*, get_*, update_*, delete_*, принимают Session.
# models.py class Product(Base): __tablename__ = "products" id = Column(Integer, primary_key=True) name = Column(String(200), nullable=False) price = Column(Float, nullable=False)

1.2. Создание моделей и записей, обратное извлечение

  • Создание таблиц — Base.metadata.create_all(bind=engine) в main.py (lifespan).
  • Создание записи (в crud.py): db.add(объект) → db.commit() → db.refresh().
  • Извлечение — db.query(Модель).filter(...).first() или .all().
# Создание def create_product(db: Session, product: schemas.ProductCreate): db_product = models.Product(**product.model_dump()) db.add(db_product) db.commit() db.refresh(db_product) return db_product # Извлечение def get_product(db: Session, product_id: int): return db.query(models.Product).filter(models.Product.id == product_id).first()

1.3. Роутинг (маршрутизация)

  • Главный файл — app/main.py: подключаются все роутеры через app.include_router().
  • Файлы роутеров — app/routers/ (auth, users, products, categories, brands, orders). Каждый содержит APIRouter() и декораторы.
  • Формирование URL — префикс из main.py (/api/v1) + префикс роутера (/products) + путь внутри роутера (/{id}).

1.4. Сохранение (транзакции)

  • db.add() — помещает объект в сессию.
  • db.commit() — фиксирует транзакцию.
  • db.refresh() — обновляет объект из БД (подгружает ID, даты).
  • При ошибке сессия закрывается без коммита (автоматический откат).

1.5. Авторизация и аутентификация (JWT)

  • Настройки — app/config.py (SECRET_KEY, ALGORITHM).
  • Генерация токена — app/routers/auth.py: функция create_access_token.
  • Получение токена — эндпоинт /auth/token (OAuth2PasswordRequestForm).
  • Проверка токена — app/dependencies.py:
    • oauth2_scheme = OAuth2PasswordBearer(tokenUrl="/auth/token")
    • get_current_user — декодирует токен, извлекает пользователя.
    • require_admin, require_admin_or_manager — проверяют роли.
# Пример защиты эндпоинта @router.post("/brands/") def create_brand( brand: schemas.BrandCreate, db: Session = Depends(get_db), current_user: schemas.User = Depends(require_admin_or_manager) ): ...

1.6. WebSocket (уведомления в реальном времени)

  • Менеджер соединений — app/websocket_manager.py (класс ConnectionManager).
  • Эндпоинт — @app.websocket("/ws/notifications") в main.py.
  • Отправка сообщения — asyncio.create_task(manager.send_message(...)) (например, после создания заказа).

⚛️ 2. Фронтенд (React + TypeScript + MUI)

2.1. Права и роли

  • Хранение роли — src/contexts/AuthContext.tsx (объект user с полем role).
  • Защита маршрутов — src/components/PrivateRoute.tsx проверяет allowedRoles.
  • Фильтрация меню — в Layout.tsx пункты меню имеют roles, и они отображаются только если роль пользователя входит в список.

2.2. Компоненты React (View)

  • Структура — src/pages/ (страницы: Login, Dashboard, Products, Categories, Brands, Users, Shop/...).
  • Переиспользуемые компоненты — src/components/ (Layout, PrivateRoute, ImageManager, NotificationSnackbar).
  • Хуки — useState, useEffect, useNavigate, useParams, кастомный useAuth.
  • API — axios-инстанс в src/api/client.ts.
  • Навигация — react-router-dom (Routes, Link, useNavigate).

2.3. WebSocket на фронтенде

  • Контекст — src/contexts/WebSocketContext.tsx создаёт соединение к ws://localhost:8000/ws/notifications для ролей admin/manager.
  • Обработка сообщений — при получении new_order генерируется событие, которое перехватывает NotificationSnackbar.

2.4. Использование MUI

  • Тема — src/theme.ts (цвета, шрифты).
  • Подключение — ThemeProvider в index.tsx.
  • Компоненты — Button, TextField, Table, Card, Drawer, Snackbar, Alert, иконки из @mui/icons-material.

🔍 3. Как быстро ориентироваться в коде

  • Найти эндпоинт → app/routers/ → нужный файл по названию → декоратор (@router.get(...)).
  • Понять модель → app/models.py (класс + поля).
  • Увидеть бизнес-логику → app/crud.py (функции create_*, get_*).
  • Аутентификация → app/routers/auth.py (генерация токена) + app/dependencies.py (проверка токена).
  • Фронтенд → src/App.tsx (маршруты), затем src/pages/ и src/components/.
  • Роли → AuthContext (хранение) + PrivateRoute (проверка).
  • WebSocket → WebSocketContext + NotificationSnackbar.
  • Используйте поиск (Ctrl+Shift+F): ищите названия функций, URL, имена моделей.

💡 4. Что важно подчеркнуть на экзамене

  • ORM — мы описываем модели на Python, SQLAlchemy преобразует их в SQL-запросы, мы не пишем сырые запросы.
  • JWT — stateless: токен подписан и содержит все данные, сервер не хранит сессии.
  • WebSocket — двунаправленная связь для push-уведомлений, в отличие от HTTP (запрос-ответ).
  • React-компоненты — функциональные, используют хуки для состояния и эффектов.
  • Роли — проверяются дважды: на бэкенде (в каждом защищённом эндпоинте) и на фронтенде (скрытие элементов и защита маршрутов).
  • MUI — ускоряет разработку за счёт готовых компонентов и кастомизации через тему.
