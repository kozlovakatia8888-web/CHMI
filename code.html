1. Бэкенд (FastAPI + SQLAlchemy)

1.1. Взаимодействие с ORM (SQLAlchemy)

app/models.py — все модели наследуются от Base (определён в database.py).

# models.py class Product(Base): __tablename__ = "products" id = Column(Integer, primary_key=True) name = Column(String(200), nullable=False) price = Column(Float, nullable=False) # relationships

app/database.py — движок, сессии, зависимость get_db.

# database.py engine = create_engine(DATABASE_URL) SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine) def get_db(): db = SessionLocal() try: yield db finally: db.close()

app/crud.py — CRUD-операции с сессией.

def get_product(db: Session, product_id: int): return db.query(models.Product).filter(models.Product.id == product_id).first()

Использование в роутере:

@router.get("/products/{product_id}") def read_product(product_id: int, db: Session = Depends(get_db)): product = crud.get_product(db, product_id) ...

1.2. Создание моделей и записей, обратное извлечение

main.py — создание таблиц при старте:

Base.metadata.create_all(bind=engine)

Создание записи (пример в crud.py):

def create_product(db: Session, product: schemas.ProductCreate): db_product = models.Product(**product.model_dump()) db.add(db_product) db.commit() db.refresh(db_product) return db_product

Извлечение: db.query(Модель).filter(...).first() или .all().

1.3. Роутинг (маршрутизация)

app/main.py — подключение роутеров:

app.include_router(auth.router, prefix="/api/v1") app.include_router(products.router, prefix="/api/v1") ...

Роутеры в app/routers/ — каждый файл с APIRouter():

router = APIRouter(prefix="/products", tags=["📦 Товары"]) @router.post("/") def create_product(...): ... @router.get("/{product_id}") def get_product(...): ...

Как формируется URL: префикс из main + префикс роутера + путь внутри роутера. Например, /api/v1/products/ и /api/v1/products/1.

1.4. Сохранение (Transaction)

  • db.add(объект) — добавляет объект в сессию.
  • db.commit() — фиксирует изменения в БД.
  • db.refresh(объект) — обновляет объект из БД (подгружает сгенерированные поля).
  • При ошибке сессия закрывается без коммита (автооткат).

1.5. Авторизация и аутентификация (JWT)

  • app/config.py — SECRET_KEY, ALGORITHM.
  • app/routers/auth.py — create_access_token и эндпоинт /auth/token.
  • app/dependencies.py — oauth2_scheme, get_current_user, require_admin, require_admin_or_manager.

Пример защиты эндпоинта:

@router.post("/brands/") def create_brand( brand: schemas.BrandCreate, db: Session = Depends(get_db), current_user: schemas.User = Depends(require_admin_or_manager) ): ...

1.6. Веб-сокеты (WebSocket)

app/websocket_manager.py — класс ConnectionManager с методами connect, disconnect, send_message.

app/main.py — эндпоинт WebSocket:

@app.websocket("/ws/notifications") async def websocket_endpoint(websocket: WebSocket): await manager.connect(websocket) try: while True: await websocket.receive_text() except WebSocketDisconnect: manager.disconnect(websocket)

Отправка уведомления после создания заказа (app/routers/orders.py):

asyncio.create_task(manager.send_message({"type": "new_order", ...}))

2. Фронтенд (React + TypeScript + MUI)

2.1. Права и роли

src/contexts/AuthContext.tsx — контекст с user (поле role).

src/components/PrivateRoute.tsx — проверяет allowedRoles, редиректит на /unauthorized.

В App.tsx маршруты обёрнуты в PrivateRoute с разными ролями:

<Route path="/admin/users" element={ <PrivateRoute allowedRoles={['admin']}> <Users /> </PrivateRoute> } />

Фильтрация меню в Layout.tsx: пункты имеют массив roles, отображаются только при совпадении с ролью пользователя.

2.2. Компоненты React (View)

  • src/pages/ — страницы: Login, Dashboard, Products, Categories, Brands, Users, Shop/ (публичные).
  • src/components/ — переиспользуемые: Layout, PrivateRoute, ShopLayout, ImageManager, NotificationSnackbar.
  • Компоненты функциональные, используют useState, useEffect, useNavigate, useParams, useAuth.
  • API-запросы через src/api/client.ts (axios-инстанс).
  • Навигация: react-router-dom — Link, useNavigate, Outlet.

2.3. WebSocket на фронтенде

src/contexts/WebSocketContext.tsx — создаёт соединение к ws://localhost:8000/ws/notifications для ролей admin/manager.

При получении сообщения типа new_order генерируется событие newOrder.

src/components/NotificationSnackbar.tsx — слушает это событие и показывает Snackbar (Material-UI).

2.4. Использование готовой библиотеки интерфейса (MUI)

  • src/theme.ts — кастомная тема MUI (цвета, шрифты, скругления).
  • src/index.tsx — ThemeProvider оборачивает приложение.
  • В компонентах импортируются готовые компоненты из @mui/material и @mui/icons-material.
  • Стилизация через проп sx или styled.

3. Как быстро ориентироваться в коде

  • Эндпоинты: app/routers/ → ищите файл по теме, смотрите декораторы.
  • Модели: app/models.py.
  • Бизнес-логика: app/crud.py.
  • Аутентификация: app/routers/auth.py + app/dependencies.py.
  • Фронтенд: начинайте с src/App.tsx, затем src/pages/ и src/components/.
  • Роли: AuthContext и PrivateRoute.
  • WebSocket: WebSocketContext и NotificationSnackbar.
  • Используйте поиск (Ctrl+Shift+F) по названиям функций, URL, классам.

4. Что важно подчеркнуть на экзамене

  • ORM — не пишем SQL, описываем модели Python, SQLAlchemy преобразует в запросы.
  • JWT — не храним сессии, токен подписан и содержит всё о пользователе.
  • WebSocket — push-уведомления в реальном времени, отличие от HTTP (запрос-ответ).
  • React-компоненты — функциональные, хуки (useState, useEffect).
  • Роли — на бэкенде (проверка в эндпоинтах) и на фронтенде (скрытие UI и защита маршрутов).
  • MUI — ускоряет разработку готовыми компонентами и кастомизацией через тему.
