🏝️ Вопрос 15. Современные архитектурные паттерны фронтенда

Islands Architecture и React Server Components (RSC) — как они решают проблемы производительности классических SPA.

🏝️ Архитектура «Изолированные острова» (Islands Architecture)

Суть: Страница генерируется как статический HTML (SSG/SSR) без JS по умолчанию. Внутри «океана» статики выделяются отдельные острова — интерактивные компоненты, которые загружают свой JS независимо. Каждый остров гидратируется сам по себе.

// Пример в Astro: компонент-остров с директивой client:load --- import Carousel from '../components/Carousel.jsx'; --- <div>Статический контент, не требующий JS</div> <Carousel client:load /> <!-- JS подгрузится только для карусели -->
Проблема SPAРешение Islands
Тяжёлый бандл при первом заходеПервый ответ — чистый HTML. JS грузится точечно, только для виджетов.
Долгая гидратация всей страницыНет глобальной гидратации. Каждый остров независим.
SEOОтличное — робот получает полный HTML.
Производительность на слабых устройствахТолько минимальный JS, не качает гигантский бандл.
TTIСтатика видна мгновенно, интерактив приходит позже, но не блокирует.
🎯 Идеальный сценарий: контентные сайты, блоги, интернет-магазины, документация — где интерактива мало, но важны скорость и SEO.
Ограничения: сложнее глобальное состояние, общение между островами (через события/шину), не подходит для приложений с интенсивным взаимодействием (Figma, Google Docs).

⚛️ React Server Components (RSC)

Суть: Расширение модели React. Компоненты делятся на:

  • Client Components — обычные React-компоненты с состоянием, хуками, DOM.
  • Server Components — выполняются только на сервере, никогда не отправляют свой код в браузер. Могут быть асинхронными, напрямую ходить в БД, читать файлы. В браузер приходит только результат рендера.
// ServerComponent.js (нет 'use client') import { db } from '@/db'; import ClientCounter from './ClientCounter'; export default async function ServerComponent() { const users = await db.user.findMany(); return ( <div> {users.map(user => <div>{user.name}</div>)} <ClientCounter initialCount={users.length} /> </div> ); }
ПроблемаРешение RSC
Загрузка ненужного JSСерверные компоненты не добавляют JS в бандл.
Проблема N+1 запросов на клиентеСерверный компонент выполняет запросы к БД, отдаёт готовые данные.
Лишние API-слоиПрямой доступ к БД из компонента, убирается «водопровод».
Hydration mismatch и задержка TTIСерверные компоненты не гидратируются, клиентских меньше.
Размер бандлаСерверные зависимости (БД, файловая система) не попадают в браузер.
🎯 Идеальный сценарий: сложные React-приложения с серверной логикой и интерактивом (e‑commerce, дашборды, CRM, соцсети).
Ограничения: серверные компоненты не используют хуки и браузерные API; требуют фреймворк (Next.js App Router, React 19); сложность потокового рендеринга.

🆚 Сравнение Islands Architecture и RSC

ХарактеристикаIslands (Astro)RSC (Next.js, React 19)
По умолчанию статикаДа, почти всё статический HTML.Нет, серверные компоненты рендерятся динамически (можно сочетать с SSG).
Маркировка островов/клиентскихclient:load'use client'
Глобальное состояниеЧерез события или внешние сторы.Контекст работает только внутри клиентского дерева.
Серверная логикаAPI-маршруты, но не из компонентов напрямую.Прямые запросы к БД, файлам из серверных компонентов.
Навигация после загрузкиОбычно полная перезагрузка (MPA-стиль).Клиентская навигация (SPA-подобно) с предзагрузкой.
ЭкосистемаAstro, возможны другие.React-экосистема, Next.js лидирует.

📌 Итог для экзамена (тезисы)

ПаттернСутьКакие проблемы решаетКогда применять
Islands ArchitectureСтатический HTML + изолированные интерактивные виджеты, каждый со своим JS.Тяжёлый бандл, долгий TTI, проблемы SEO из-за JS.Контентные сайты (блоги, документация, каталоги).
React Server ComponentsСерверные компоненты (без JS в браузере) + клиентские.Ненужный JS, избыточные API-запросы, водопровод данных, большая гидратация.Сложные React-приложения с серверной логикой (e‑commerce, дашборды, соцсети).
💡 Главный вывод: оба паттерна отказываются от монолитного клиентского JS и переносят рендер/логику на сервер. Islands делают это просто и жёстко, RSC — глубоко интегрированно в React, позволяя смешивать серверный и клиентский код. Оба значительно улучшают производительность по сравнению с классической SPA.
