🏝️ Современные архитектурные паттерны фронтенда: Islands Architecture и React Server Components (RSC)

Как решают проблемы производительности классических SPA: тяжёлый бандл, долгий TTI, плохой SEO.

Islands Architecture (на примере Astro)

Статический HTML по умолчанию + изолированные интерактивные "острова", каждый со своим JS. Острова загружаются независимо (client:load, client:idle, client:visible).

--- import Carousel from '../components/Carousel.jsx'; --- <div>Статический контент</div> <Carousel client:load /> <!-- JS только для карусели -->
  • ✅ Мгновенный TTV, отличное SEO, точечная гидратация.
  • ✅ Нет глобального JS-бандла.
  • ❌ Сложная коммуникация между островами, не подходит для сложных SPA.

React Server Components (RSC)

Компоненты делятся на серверные (выполняются на сервере, без JS в браузере) и клиентские ('use client'). Серверные компоненты могут быть асинхронными, напрямую ходить в БД.

// ServerComponent.js import { db } from '@/db'; import ClientCounter from './ClientCounter'; export default async function ServerComponent() { const users = await db.user.findMany(); return <div>{users.map(u => <div>{u.name}</div>)}<ClientCounter /></div>; }
  • ✅ Серверный код не попадает в бандл, отличная производительность, прямая работа с БД.
  • ✅ Постепенная миграция, стриминг (Suspense).
  • ❌ Требует фреймворка (Next.js App Router), серверные компоненты не имеют хуков/состояния.

Сравнение Islands и RSC

ХарактеристикаIslands (Astro)RSC (Next.js)
Статика по умолчаниюДаНет (динамика + SSG опционально)
Маркировка интерактиваclient:load'use client'
Серверная логикаAPI-маршрутыПрямые запросы из компонентов
НавигацияMPA-стиль (перезагрузка)SPA-подобная (клиентский роутинг)
🎯 Вывод: оба паттерна отказываются от монолитного JS, перенося рендер на сервер. Islands — для контентных сайтов, RSC — для сложных React-приложений с серверной логикой.
