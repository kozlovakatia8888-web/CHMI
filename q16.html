🔒 Безопасность веб-приложений: XSS, CSRF, SQL-инъекции — углублённо

Механизмы атак и встроенная защита в современных фреймворках (экранирование, CSRF-токены, параметризованные запросы).

1. XSS (Cross-Site Scripting)

Суть: внедрение вредоносного JavaScript в веб-страницу, выполняющегося в браузере жертвы. Типы: отражённый (Reflected), хранимый (Stored), DOM-based.

Пример отражённого XSS:
https://site.com/search?q=<script>alert(document.cookie)</script> — если сервер выводит параметр без экранирования.
Защита (встроенная во фреймворки):
- Автоматическое экранирование в шаблонах (React JSX, Vue {{ }}, Django {{ var }}, Jinja).
- Content Security Policy (CSP).
- HttpOnly cookie (защита от кражи сессионных кук).
⚠️ Опасно: v-html, dangerouslySetInnerHTML, |safe — отключают защиту!

2. CSRF (Cross-Site Request Forgery)

Суть: принуждение аутентифицированного пользователя отправить поддельный запрос на уязвимый сайт (перевод денег, смена пароля).

Классический пример:
<img src="http://bank.com/transfer?to=hacker&amount=1000" width="0" height="0"> — при загрузке страницы на evil.com браузер отправляет GET-запрос с куками жертвы.
Защита (встроенная во фреймворки):
- CSRF-токены (синхронизатор токенов) — Django {% csrf_token %}, Spring Security, Rails.
- SameSite атрибут cookie (SameSite=Lax или Strict) — современный стандарт.
- Проверка Referer/Origin (дополнительный барьер).
💡 CSRF-токены генерируются сервером, вставляются в формы и проверяются при отправке. Атакующий сайт не может узнать токен жертвы.

3. SQL-инъекции (SQL Injection)

Суть: внедрение SQL-кода в параметры запроса, который неправильно конкатенируется в SQL-запрос. Позволяет читать, изменять, удалять данные.

Уязвимый код:
query = f"SELECT * FROM users WHERE name = '{user_input}'"
При вводе ' OR '1'='1 возвращаются все пользователи.
'; DROP TABLE users; -- — удаление таблицы.
Защита (золотой стандарт):
- Параметризованные запросы (prepared statements) — код и данные разделены.
- Примеры: cursor.execute("SELECT * FROM users WHERE name = %s", (user_input,)) (Python).
- ORM (Hibernate, SQLAlchemy, Django ORM, EF) — используют параметризованные запросы по умолчанию.
- Никогда не конкатенировать пользовательский ввод!
