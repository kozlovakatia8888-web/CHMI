📄 Стандарт OpenAPI и Swagger — углублённо

Спецификация OpenAPI (YAML/JSON) и экосистема инструментов Swagger: Editor, UI, Codegen.

📌 Что такое OpenAPI?

OpenAPI (ранее Swagger Specification) — это стандарт описания REST API в машиночитаемом формате (YAML или JSON).

Что описывает: endpoint'ы, параметры (path, query, header, cookie), схемы данных, аутентификацию, коды ответов.

openapi: 3.0.0 info: title: User API version: 1.0.0 paths: /users/{id}: get: parameters: - name: id in: path schema: type: integer responses: 200: description: OK content: application/json: schema: $ref: '#/components/schemas/User'
🎯 Главная ценность: единый источник правды (single source of truth) для API.

🛠️ Экосистема Swagger

  • Swagger Editor — веб-редактор для написания и валидации OpenAPI-файлов с предпросмотром документации.
  • Swagger UI — интерактивная документация с возможностью выполнять реальные запросы (Try it out). Часто хостится по пути /swagger-ui.
  • Swagger Codegen — генератор клиентских SDK (Python, Java, TypeScript, C# и др.) и серверных скелетонов из OpenAPI.
# Пример генерации TypeScript-клиента (axios) swagger-codegen generate -i api.yaml -l typescript-axios -o ./generated-client

Дополнительные инструменты:

  • SwaggerHub — платформа для командной работы над OpenAPI (хостинг, версионирование).
  • Swagger Inspector — тестирование API с экспортом в OpenAPI.

📋 Практические сценарии

  • Design-first: сначала описываем API в OpenAPI, затем генерируем код.
  • Автоматическая документация: Swagger UI показывает всегда актуальную документацию.
  • Генерация клиента для микросервисов: сервис А публикует OpenAPI, сервис Б генерирует клиент.
  • Контрактное тестирование: спецификация — истина для проверки соответствия реального API.
