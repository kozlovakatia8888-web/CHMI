📘 Вопрос 1. Архитектурные парадигмы веб-фронтенда: MPA, SPA, SSR, SSG

Полный разбор: как работают, в чём различия рендеринга, SEO и времени до интерактивности (TTI). С примерами и тестами.

🌐 Что такое архитектура веб-приложения?

Архитектура определяет, где и как генерируется HTML-код, который видит пользователь, и когда загружается JavaScript для интерактивности. От этого зависят скорость загрузки, SEO, поведение при навигации.

📄 MPA Multi-Page Application

Как работает: каждый переход по ссылке — новый запрос к серверу. Сервер полностью генерирует HTML-страницу и отправляет её. Браузер перезагружает страницу целиком.

Рендеринг: серверный (каждый раз).

SEO: отличное — робот получает готовый контент.

TTI (Time to Interactive): при первом визите — быстро (HTML уже содержит контент). При навигации — медленно, т.к. полная перезагрузка.

Где применяется: блоги, новостные порталы, корпоративные сайты, интернет-магазины на CMS (WordPress, Tilda).

📌 Пример фреймворков: Django, Ruby on Rails, Laravel, WordPress.
⚡ SPA Single-Page Application

Как работает: при первом визите скачивается «пустая» HTML-обёртка и огромный JS-бандл. Дальше роутинг на клиенте — JS подгружает данные через API и перерисовывает страницу без перезагрузки.

Рендеринг: клиентский (браузер).

SEO: плохое (из коробки) — робот видит пустой div. Нужен рендеринг на сервере (SSR) или prerendering.

TTI: первый визит — долго (загрузка и выполнение JS). Последующие навигации — мгновенно.

Где применяется: дашборды, CRM, онлайн-редакторы (Google Docs, Figma, Trello).

📌 Примеры: React (create-react-app), Vue CLI, Angular.
🖥️ SSR Server-Side Rendering

Как работает: первый запрос — сервер генерирует полный HTML. Затем загружается JS и происходит гидратация — статическая верстка оживает, становится интерактивной. Дальше навигация как в SPA.

Рендеринг: сервер (первый раз), потом клиент.

SEO: отличное.

TTI: пользователь видит контент быстро (TTV — Time to View), но кликнуть может только после гидратации — задержка.

Где применяется: e-commerce, новостные порталы с виджетами, сайты, где нужны и SEO, и интерактивность.

📌 Примеры: Next.js (React), Nuxt (Vue), Remix.
📦 SSG Static Site Generation

Как работает: HTML-страницы генерируются один раз на этапе сборки (build time). Сервер просто отдаёт готовые файлы через CDN. Нет серверной обработки запросов.

Рендеринг: заранее (build-time).

SEO: идеальное.

TTI: максимально быстрый — никакой задержки, страницы лёгкие.

Где применяется: блоги, документация, лендинги, Jamstack-сайты.

📌 Примеры: Hugo, Jekyll, Astro, Next.js (export), 11ty.

🧠 Ключевые метрики: TTI, TTV, гидратация

  • TTV (Time to View) — когда пользователь видит контент (пусть даже статический).
  • TTI (Time to Interactive) — когда можно кликать, вводить текст, взаимодействовать.
  • Гидратация (hydration) — процесс, когда серверный HTML «привязывается» к клиентскому JS (вешаются обработчики, восстанавливается состояние).

⚙️ Гибридные подходы (ISR, Partial Hydration)

ISR (Incremental Static Regeneration) — улучшенный SSG: страницы обновляются фоном после сборки (Next.js).
Partial Hydration / Islands Architecture — гидратируется только часть страницы (Astro).
React Server Components — компоненты рендерятся на сервере и не отправляют JS клиенту.
