🌐 Сетевые протоколы для API: REST, GraphQL, gRPC

Сравнение, сильные и слабые стороны, сценарии использования.

REST

HTTP-методы (GET, POST, ...), ресурсы через URL, stateless, обычно JSON.

  • ✅ Простота, кеширование, OpenAPI.
  • ❌ Over-fetching, under-fetching, много запросов.

GraphQL

Единый endpoint, клиент запрашивает нужные поля в одном запросе.

query { user(id: 123) { name orders { total } } }
  • ✅ Нет over-fetching, один запрос, сильная типизация.
  • ❌ Сложный на сервере, плохое кеширование, риск глубоких запросов.

gRPC

HTTP/2, Protocol Buffers (бинарный), строгая типизация, стриминг.

service UserService { rpc GetUser(GetUserRequest) returns (User); }
  • ✅ Высокая производительность, стриминг, генерация кода.
  • ❌ Сложность отладки, не поддерживается браузерами напрямую.
