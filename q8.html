📦 Сериализация данных: бинарная (pickle, Java Serialization) vs текстовая (JSON, XML)

Сравнение, преимущества, недостатки, безопасность и сценарии использования.

Бинарная сериализация (pickle, Java Serialization)

Компактный, быстрый, сохраняет граф объектов и циклические ссылки.

import pickle data = {"name": "Alice"} with open("data.pkl", "wb") as f: pickle.dump(data, f)
⚠️ Критическая уязвимость: десериализация из недоверенного источника может привести к выполнению произвольного кода (RCE). Никогда не используйте pickle.load() на данных от пользователя!
  • Плюсы: компактность, скорость, сохранение типов и циклических ссылок.
  • Минусы: небезопасность, языкозависимость, проблемы с версиями, нечитаемость.

Текстовые форматы (JSON, XML)

// JSON {"name":"Alice","scores":[100,95]} // XML <person><name>Alice</name><scores>...</scores></person>
  • Плюсы: человекочитаемость, языконезависимость, безопасность (стандартный парсинг не выполняет код).
  • Минусы: избыточность (большой размер), медленнее, нет циклических ссылок, ограниченные типы.
