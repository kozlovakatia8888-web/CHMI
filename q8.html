📦 Вопрос 8. Сериализация данных

Бинарная (pickle, Java Serialization) vs текстовая (JSON, XML) — сравнение, недостатки, безопасность и сценарии использования.

📌 Что такое сериализация

Сериализация — превращение сложного объекта (с данными, ссылками, типами) в последовательность байтов или строку для сохранения на диск, передачи по сети или межпроцессного взаимодействия. Десериализация — обратное восстановление объекта.

🧬 Бинарная сериализация (pickle, Java Serialization)

Встроенные механизмы: java.io.Serializable, pickle в Python, BinaryFormatter в C#. Сохраняют объект в компактном бинарном виде, включая типы, структуру и граф объектов (ссылки).

import pickle data = {"name": "Alice", "scores": [100, 95]} with open("data.pkl", "wb") as f: pickle.dump(data, f) # бинарная запись with open("data.pkl", "rb") as f: restored = pickle.load(f) # восстановление

✅ Плюсы

  • Компактность — размер меньше, чем у JSON/XML.
  • Скорость — быстрее парсится и собирается.
  • Сохраняет граф объектов (один объект не дублируется).
  • Поддерживает циклические ссылки.
  • Сохраняет полную информацию о типе (удобно для кэша внутри приложения).

❌ Недостатки и риски

  • Небезопасность (главный минус!) — десериализация из недоверенного источника может привести к выполнению произвольного кода (RCE). Пример: pickle и Java Serialization известны цепочками гаджетов.
  • Языкозависимость — файл одного языка не прочитать из другого.
  • Проблемы с версиями — изменение класса ломает совместимость.
  • Нечитаемость — нельзя открыть и понять в текстовом редакторе.
⚠️ Критически опасно! Никогда не десериализуйте данные от незнакомцев через pickle или Java Serialization. Это одна из самых опасных уязвимостей веб-приложений.

📄 Текстовые форматы (JSON, XML)

Человекочитаемые, языконезависимые.

// JSON {"name":"Alice","scores":[100,95]} // XML <person><name>Alice</name><scores><score>100</score><score>95</score></scores></person>

✅ Плюсы

  • Человекочитаемость — можно открыть, понять, поправить вручную.
  • Языконезависимость — парсится любым языком.
  • Безопасность (относительная) — стандартный парсинг не выполняет произвольный код.
  • Поддержка схем (JSON Schema, XSD) для валидации.
  • JSON — стандарт для веб-API, конфигураций.

❌ Недостатки

  • Избыточность — большой объём (особенно XML).
  • Медленнее парсинг из-за синтаксического разбора строк.
  • Нет поддержки графов объектов (циклические ссылки невозможны).
  • Ограниченные типы (дата, бинарные данные требуют base64).
  • XML отдельно — очень многословный и медленный.

🧨 Особые недостатки и итоговые рекомендации

🔴 Бинарная сериализация — критическая опасность!
Атака «десериализация гаджетов» позволяет выполнить любую команду на сервере. Современные рекомендации: не использовать pickle/Java Serialization для обмена данными между доверенными границами. Вместо них — JSON, Protocol Buffers, MessagePack, или безопасные альтернативы.
💡 Рекомендации по выбору:
• Для веб-API → JSON (стандарт).
• Для конфигураций → JSON или YAML.
• Для внутреннего кэша в одном приложении (безопасные данные) → бинарная сериализация допустима, но лучше рассмотреть Protocol Buffers, MessagePack или просто JSON.
• Никогда не используйте pickle.load() или ObjectInputStream на данных от пользователя или по сети!
