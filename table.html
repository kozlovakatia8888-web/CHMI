📚 Шпаргалка по HCI

Основные понятия и сводная таблица тезисов по всем 20 вопросам — для быстрого повторения перед экзаменом.

🔑 Ключевые термины

ТерминОпределение
DOMДревовидное представление HTML-документа, доступное для JavaScript.
MPAMulti‑Page Application — традиционная архитектура с перезагрузкой страниц при каждом действии.
SPASingle‑Page Application — один HTML-скелет, роутинг на клиенте, подгрузка данных по API.
SSRServer‑Side Rendering — генерация HTML на сервере с последующей гидратацией на клиенте.
SSGStatic Site Generation — предварительная сборка HTML на этапе билда, отдача через CDN.
ГидратацияПроцесс «оживления» статического HTML клиентским JavaScript (привязка событий).
TTITime to Interactive — время, когда страница становится полностью интерактивной.
TTVTime to View — время, когда пользователь видит контент (до интерактивности).
Event BubblingВсплытие событий — фаза движения события от целевого элемента к корню.
Event CapturingЗахват событий — фаза движения от корня к целевому элементу.
ДелегированиеОдин обработчик на родителе, обрабатывающий события от всех детей за счёт всплытия.
Закон ФиттсаВремя достижения цели зависит от расстояния и размера цели: T = a + b·log₂(D/W+1).
Закон ХикаВремя выбора растёт логарифмически с числом альтернатив: RT = a + b·log₂(n+1).
GOMSМодель Goals, Operators, Methods, Selection rules для когнитивно-моторного анализа.
KLMKeystroke-Level Model — упрощённый расчёт времени через K, P, B, M, H операторы.
Когнитивная нагрузкаОбъём умственной работы, требуемый для выполнения задачи; бывает внешняя (интерфейс) и внутренняя (сложность).
Закон МиллераЧеловек удерживает 7±2 (5–9) объектов в рабочей памяти одновременно.
SUSSystem Usability Scale — опросник из 10 пунктов, дающий оценку юзабилити от 0 до 100.
SEQSingle Ease Question — один вопрос о лёгкости выполнения задачи (шкала 1–5/7/9).
ORMObject-Relational Mapping — маппинг объектов ООП на реляционные таблицы.
N+1 проблемаПри ленивой загрузке: 1 запрос на родителей + N запросов на детей. Решается жадной загрузкой.
pickleБинарная сериализация в Python; небезопасна для данных от пользователя (RCE).
JWTJSON Web Token — stateless токен с подписью, содержит данные пользователя и срок истечения.
СессииStateful подход: сервер хранит сессию, клиент получает Session ID в cookie.
OAuth 2.0Протокол делегирования доступа (авторизации) без передачи пароля.
OpenAPIСтандарт описания REST API в YAML/JSON (машиночитаемый).
Swagger UIИнтерактивная документация с возможностью выполнять запросы (Try it out).
gRPCВысокопроизводительный RPC-фреймворк на HTTP/2 и Protocol Buffers, поддерживает стриминг.
GraphQLЯзык запросов для API, позволяющий клиенту запрашивать только нужные поля за один запрос.
WCAGWeb Content Accessibility Guidelines — стандарт доступности (POUR).
Shadow DOMИнкапсуляция стилей и DOM для Web Components.
Module FederationПлагин Webpack для динамической загрузки модулей между приложениями (микрофронтенды).
MiddlewareФункции в конвейере обработки HTTP-запроса (логирование, аутентификация, CORS и др.).
Dependency InjectionПаттерн, при котором объект получает зависимости извне (конструктор), а не создаёт их сам.
F-паттернТраектория сканирования страницы: две горизонтальные линии вверху и вертикаль по левому краю.
Эффект последовательной позицииЛучшее запоминание первых (primacy) и последних (recency) элементов списка.

📋 Сводка по 20 вопросам — основные тезисы

№ТемаКлючевые тезисы
1Архитектурные парадигмыMPA (сервер, SEO), SPA (клиент, плохое SEO), SSR (гидратация), SSG (build-time, CDN). TTI — время до интерактивности.
2DOM и событияDOM — дерево. Фазы: capture → target → bubble. Делегирование — один обработчик на родителе.
3Законы Фиттса и ХикаФиттс: T = a + b·log₂(D/W+1) — важны размер и расстояние. Хика: RT = a + b·log₂(n+1) — группируйте варианты.
4GOMS и KLMGOMS — цели, операторы, методы, правила. KLM — упрощённый расчёт времени (K, P, B, M, H).
5Когнитивная нагрузкаЗакон Миллера (7±2). Стратегии: группировка, прогрессивное раскрытие, узнавание вместо воспоминания.
6Критерии юзабилитиСкорость, ошибки, обучение, удовлетворение. SUS (0–100, норма 68), SEQ (после задачи).
7ORMОбъектно-реляционное несоответствие. Антипаттерны: N+1, God Object, массовые операции через цикл.
8СериализацияБинарная (pickle) — компактно, но небезопасно (RCE). JSON — человекочитаем, везде. XML — избыточен.
9Аутентификация, сессии, JWTИдентификация → аутентификация → авторизация. Сессии (stateful) — отзыв, HttpOnly; JWT (stateless) — масштабируемость, но нельзя отозвать.
10OAuth 2.0Роли: владелец, клиент, сервер авторизации, сервер ресурсов. Authorization Code Flow — код + секрет → токен.
11OpenAPI и SwaggerOpenAPI — спецификация (YAML/JSON). Swagger Editor — написание, UI — документация с Try it out, Codegen — генерация клиентов.
12REST, GraphQL, gRPCREST — простой, кеширование. GraphQL — гибкость, один запрос. gRPC — бинарный, стриминг, микросервисы.
13Дизайн-системы и WCAGДизайн-система: токены, компоненты, паттерны. WCAG: Perceivable, Operable, Understandable, Robust. Контраст ≥4.5:1, ARIA, клавиатура.
14Визуальная композиция и цветЦелостность, доминанта, уравновешенность. Правило 60-30-10: фон, вторичный, акцентный.
15Islands и RSCIslands (Astro): статика + изолированные виджеты. RSC (Next.js): серверные компоненты без JS в бандле, прямая работа с БД.
16Безопасность (XSS, CSRF, SQLi)XSS → экранирование (фреймворки). CSRF → CSRF-токены, SameSite. SQLi → параметризованные запросы/ORM.
17Микрофронтендыiframe (изоляция), Web Components (Shadow DOM), Module Federation (общие зависимости, синглтоны).
18Middleware и DIMiddleware — конвейер (логирование, аутентификация). DI — внедрение зависимостей через конструктор → тестируемость, гибкость.
19ТипографикаSerif (статьи), Sans‑serif (UI), Monospace (код). Размер ≥16px, длина 50–75 символов, line-height 1.5, контраст ≥4.5:1.
20Восприятие и вниманиеF-паттерн: важное — вверху и слева. Primacy/recency: первые и последние элементы запоминаются лучше середины.
