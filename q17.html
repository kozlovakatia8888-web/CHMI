🧩 Микрофронтенды: архитектура, способы интеграции и решаемые проблемы

iframe, Web Components, Module Federation — сравнение, плюсы, минусы, сценарии.

🔍 Решаемые проблемы

  • Масштабирование разработки: разные команды владеют своими областями (каталог, корзина, личный кабинет), нет "merge hell".
  • Независимое развёртывание: обновление одного микрофронтенда без пересборки всего приложения.
  • Технологическая гетерогенность: можно использовать разные фреймворки (React, Vue, Svelte) на одной странице.
  • Изоляция ошибок: падение одного микрофронтенда не "роняет" всю страницу.
  • Ускорение загрузки: пользователь получает только нужные части (ленивая загрузка).

🧩 Способы интеграции

1. iframe

Самый старый способ, полная изоляция CSS/JS. Обмен данными через postMessage.

  • ✅ Полная изоляция, не требует специальной сборки, можно встроить любое legacy-приложение.
  • ❌ Проблемы с SEO, навигацией (история браузера), размером, дублирование зависимостей, плохая производительность.

2. Web Components (Custom Elements + Shadow DOM)

Стандартный подход, каждый микрофронтенд — кастомный HTML-элемент с изолированным DOM и стилями.

  • ✅ Родной стандарт, изоляция стилей (Shadow DOM), независимость от фреймворков, легко интегрировать.
  • ❌ Поддержка браузеров (полифиллы), Shadow DOM мешает глобальным стилям, сложный SSR, дублирование зависимостей.

3. Module Federation (Webpack 5)

Современный способ: динамическая загрузка модулей из других приложений во время выполнения. Общие библиотеки (React, lodash) — синглтоны.

// webpack.config.js (хост) new ModuleFederationPlugin({ name: 'host', remotes: { cart: 'cart@http://localhost:3001/remoteEntry.js' }, shared: { react: { singleton: true }, 'react-dom': { singleton: true } } });
  • ✅ Нулевое дублирование зависимостей, производительная ленивая загрузка, бесшовная навигация, общее состояние (Redux).
  • ❌ Привязка к Webpack (есть аналоги для Vite), сложная настройка, возможны проблемы с версиями общих библиотек.
🎯 Современный выбор: Module Federation для новых проектов, Web Components для максимальной совместимости, iframe — только для сторонних виджетов (платёжные формы).
